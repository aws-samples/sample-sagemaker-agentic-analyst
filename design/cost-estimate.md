# コスト試算

## 前提条件

### ユーザー規模

| パラメータ                    | 値             | 根拠                                                                 |
| ----------------------------- | -------------- | -------------------------------------------------------------------- |
| 登録ユーザー数                | 50人           | [AWS CDP試算例](https://aws.amazon.com/jp/cdp/genai-chat-app/)に準拠 |
| カタログ登録テーブル数        | 100テーブル    | 指定                                                                 |
| DAU（日次アクティブユーザー） | 50人           | 全ユーザーが平日に利用する想定                                       |
| 1人あたりの利用頻度           | 10回/日        | CDP試算例に準拠                                                      |
| 月間稼働日数                  | 22日           | 平日のみ                                                             |
| リージョン                    | ap-northeast-1 | 実環境                                                               |

### ユーザー行動モデル

データ基盤のリアルなユースケースとして、以下の3ペルソナを想定。利用頻度10回/日の内訳：

| ペルソナ                           | 人数 | 主な行動                                    | チャット回数/日 | 直接クエリ回数/日 |
| ---------------------------------- | ---- | ------------------------------------------- | --------------- | ----------------- |
| データコンシューマー（営業分析）   | 35人 | AIエージェントに質問、定型レポート確認      | 7回             | 3回               |
| データプロデューサー（データ管理） | 10人 | カタログ管理、データ品質確認、Publish/Share | 3回             | 7回               |
| ドメイン管理者（セキュリティ監査） | 5人  | 監査ログ検索、アクセス異常検知              | 2回             | 8回               |

### 1チャットセッションあたりのリソース消費モデル

| メトリクス                  | 値            | 導出方法                                                             |
| --------------------------- | ------------- | -------------------------------------------------------------------- |
| LLMターン数/チャット        | 3ターン       | エージェントが検索→詳細取得→回答の3段階（内部ループ）                |
| 入力トークン/ターン         | 4,600トークン | システムプロンプト(1K) + ツール定義(2K) + 会話コンテキスト(1.6K平均) |
| 出力トークン/ターン         | 570トークン   | thinking(300) + response/tool_call(270)の3ターン平均                 |
| ツール呼び出し/チャット     | 2回           | catalog_search(1) + athena_query(1) or s3_read(1)                    |
| Athenaスキャン量/クエリ     | 5 MB          | 100テーブル × 平均50MB/テーブル、列プルーニング適用                  |
| DataZone APIコール/チャット | 4件           | SearchListings(1) + ListSubscriptions(1) + GetListing(2)             |

**導出根拠**:

- システムプロンプト: 実測2,145文字 ≈ 1,000トークン（日本語混在）
- ツール定義: MCP 4ツール + code_interpreter ≈ 2,000トークン（JSON Schema）
- 会話コンテキスト: ターンごとに積み上がる履歴の3ターン平均。Turn1: 100, Turn2: 1,600, Turn3: 3,100 → 平均1,600
- 出力: Turn1: thinking(300)+tool_call(50)=350, Turn2: 350, Turn3: thinking(500)+response(500)=1,000 → 平均570
- DataZone API使用量パターン: 開発環境の実測でDataZone APIの課金リクエスト数・コンピュートユニット消費量を観測し、定常運用時の水準を推定

### 仮定と不確実性

| 項目                    | 仮定                           | 不確実性                                                            | 影響度                 |
| ----------------------- | ------------------------------ | ------------------------------------------------------------------- | ---------------------- |
| AgentCore RT I/O wait率 | 70%（公式Pricing Example準拠） | エージェントの処理パターンに依存                                    | 低（RT自体が低コスト） |
| Athenaスキャン量        | 5MB/クエリ                     | パーティショニング・列プルーニングで大幅に変動                      | 低                     |
| 無料枠                  | SageMaker Catalog無料枠を控除  | アカウント単位の枠。共有アカウントでは消費済みの可能性（+$1.13/月） | 低                     |
| Cognito無料枠           | 控除しない                     | Always Freeだがアカウント共有のため保守的に見積                     | 低                     |

## サービス別コスト試算

### 1. Bedrock（LLM推論）

使用モデル: Claude Sonnet 4.6（`global.anthropic.claude-sonnet-4-6`、cross-region inference、追加料金なし）

| 項目               | 計算                                                 | 月額       |
| ------------------ | ---------------------------------------------------- | ---------- |
| 入力トークン       | 86.5M tokens × $3.00/M                               | **$259.5** |
| 出力トークン       | 6,270チャット × 3ターン × 570 = 10.7M tokens × $15/M | **$160.5** |
| **Bedrock合計**    |                                                      | **$420.0** |

> Extended thinkingを使用しているため、出力トークンにはthinkingトークンが含まれる（出力トークンとして課金）。

### 2. コンピュート（AgentCore Runtime + Lambda）

**AgentCore Runtime**: 実際のリソース消費量で課金。LLM応答待ち（I/O wait）の間はCPU課金ゼロ（[公式Pricing](https://aws.amazon.com/bedrock/agentcore/pricing/)）。1チャットセッション: 60秒、70% I/O wait → CPU active 18秒、メモリ ピーク1GB。

| 項目                       | 計算                                                            | 月額      |
| -------------------------- | --------------------------------------------------------------- | --------- |
| AgentCore RT CPU           | 6,270セッション × 18秒 × 1vCPU × ($0.0895/3600)                 | **$2.81** |
| AgentCore RT メモリ        | 6,270セッション × 60秒 × 1GB × ($0.00945/3600)                  | **$0.99** |
| webapp Lambda (1024MB)     | 55,000リクエスト × 1024MB × 500ms = 28,160 GB-s × $0.0000166667 | **$0.47** |
| Tool Lambda 4関数 (256MB)  | 12,540呼び出し × 256MB × 2s = 6,420 GB-s × $0.0000166667        | **$0.11** |
| **コンピュート合計**       |                                                                 | **$4.38** |

### 4. SageMaker Catalog（DataZone）

料金ページ（[aws.amazon.com/sagemaker/pricing](https://aws.amazon.com/sagemaker/pricing/)）に基づく。

**課金対象APIと無料APIの区分**:

- 無料API: `Search`, `CreateDomain`, `CreateProject`, `GetEnvironmentCredentials`, `ListNotifications`, プロジェクト/ユーザー/ポリシー管理系
- 課金対象API: `SearchListings`, `ListSubscriptions`, `GetListing`, `GetAsset`, `CreateDataSource`, `StartDataSourceRun` 等

| 項目                      | 計算                                                                                           | 月額      |
| ------------------------- | ---------------------------------------------------------------------------------------------- | --------- |
| リクエスト                | エージェント: 25,080件 + SMUS UI: 11,000件 → 36,000件。無料枠4,000件控除 → 32,000件 × $10/100K | **$3.20** |
| メタデータストレージ      | 100テーブル × ~200KB = ~20MB（無料枠内）                                                       | **$0**    |
| コンピュート              | 月次データソース差分同期 ~0.5 CU。無料枠0.2 CU控除 → 0.3 CU × $1.776                           | **$0.53** |
| **SageMaker Catalog合計** |                                                                                                | **$3.73** |

**SMUS環境自体のインフラ固定費**: SMUSドメイン（DataZoneドメイン + IAMロール群 + WebポータルUI）にインフラ固定費は発生しない。このアプリで使用するブループリント（LakeHouseDatabase, Tooling）はGlue DB・Lake Formation権限・IAMロール等のみを作成し、課金リソースを生成しない。OpenSearch（KnowledgeBaseブループリント）、MLflow（MLExperimentsブループリント）、Redshift等の高額リソースは対応ブループリントを有効にしない限り作成されない。
