# E2Eテストガイド

このドキュメントでは、Agentic AnalystのE2Eテストシナリオを定義します。

**対象ペルソナ:** 全ペルソナ

**Phase 5:** webapp操作による動作検証

**前提:** [01-deployment.md](./01-deployment.md) と [02-sagemaker-config.md](./02-sagemaker-config.md) が完了していること

## 概要

AIエージェント（playwright-cli）がWebアプリを操作して実施するE2Eテストの合格条件を定義します。

**スコープ:** [requirements.md](../design/requirements.md) のストーリー#1〜#9を対象。

- ストーリー#1〜#3（SMUS操作）: S-SMUSシナリオ
- ストーリー#4〜#9（Webアプリ）: S0〜S3シナリオ

## 既知の未解決問題

現時点で未解決の問題はありません。

## テストデータ

### 構造化データ（Athena/Lake Formation）

| テーブル                      | 件数  | 主要カラム                                                                        | 検証用値の例                |
| ----------------------------- | ----- | --------------------------------------------------------------------------------- | --------------------------- |
| store_details                 | 5,999 | store_id, store_name, store_city, store_state                                     | `S001`, `Downtown Flagship` |
| retail_sales_performance      | 5,000 | date, store_id, product_id, sales_amount, units_sold                              | `P101`, `15200.50`          |
| ecommerce_customer_behavior   | 5,000 | date, customer_id, product_id, page_views, purchase_amount                        | `C1001`, `89.99`            |
| sales_rep_performance（機密） | 5,000 | date, sales_rep_id, total_sales_amount, deals_closed, customer_satisfaction_score | `REP001`, `4.8`             |
| b2b_sales_pipeline（機密）    | 5,000 | date, sales_rep_id, customer_id, deal_value, pipeline_stage                       | `Negotiation`, `120000.00`  |

### 非構造化データ（S3 Access Grants）

| パス                                          | 内容         | 検証キーワード               |
| --------------------------------------------- | ------------ | ---------------------------- |
| `unstructured/public/product_catalog.txt`     | 製品カタログ | `CATALOG-2025-VERIFIED`      |
| `unstructured/confidential/strategy_2025.txt` | 戦略文書     | `STRATEGY-2025-CONFIDENTIAL` |

> **注意:** テストファイルはCDKデプロイ時に `BucketDeployment` で自動アップロード済みです

## 前提条件

### インフラ

- CDKデプロイ完了済みで、CloudFront経由でWebアプリにアクセス可能（`https://<FrontendDomainName>`）

### SMUS設定（ランブックで事前実施）

- SMUSドメイン作成済み
- 3ユーザー（dg-corp-admin, dg-data-owner, dg-business-analyst）がCognito経由でログイン可能
- サブスクリプション設定:
  - dg-data-owner: 全テーブル
  - dg-business-analyst: 一般テーブルのみ（store_details, retail_sales_performance, ecommerce_customer_behavior）

## プロジェクト選択（全シナリオ共通・必須）

ログイン後、**Athenaクエリ・AIエージェントを使用する前に、必ず画面左下のプロジェクトセレクターでプロジェクトを選択してください。** プロジェクト未選択の状態でクエリやエージェントを実行するとエラーになります。

| ユーザー              | 選択するプロジェクト                |
| --------------------- | ----------------------------------- |
| `dg-business-analyst` | `analyst-data-access`               |
| `dg-data-owner`       | `demo-salesdb-project`              |
| `dg-corp-admin`       | （任意 — いずれもメンバーではない） |
